Building a Santa tracker⌗

Introduction⌗

Since I was a little kid I’ve always been very interested in computers and electronics. The very first computer I had was a really crappy laptop that couldn’t play any videogames so the only thing I could do on it was browsing the web, and on Christmas Eve I was always greeted by Christmas themed Google Doodles, these little things always felt special, especially during Christmas since they always showed a map of Santa Claus movements. Today I will explain the logic behind the Santa tracker and we will build together a small Python script that shows Santa’s movements in real time.

Reverse-engineering time⌗

The first thing I did was searching for a Wikipedia article about it, and I actually found it! The crazy thing I found out is that the code for the website is on Github.

I looked into the repository and I found a firebase.js with some configuration settings, from there I decided that it was much easier to use the browser devtools and just look for network traffic from Firebase.

I found this URL that looked promising:

https://firebasestorage.googleapis.com/v0/b/santa-api.appspot.com/o/route-2025%2Fsanta_en.json?2021=&alt=media

The URL returns an enormous JSON file, that actually contains a list of coordinates and timestamps with various other informations, I actually found the route!

To be honest I initially thought that the logic behind this applications was a little more complex, since you don’t want a kid to see that Santa is over his house at 2:00 PM on Christmas Eve. The easy fix to this problem is just careful route planning, the developers just needed to input a route that matched the new day line as it moved so that Santa would be in the right place at the exact time.

Building our tracker⌗

Decoding the JSON⌗

The JSON file is just a big list of objects, let’s see the one for Rome:

"id": "rome", // the object id is just the city name in lowercase
"arrival": 1577226180000, // time of arrival in unix timestamp
"departure": 1577226240000, // time of departure in unix timestamp
"population": 2318895, // city population
"presentsDelivered": 4761568882, // counter for presents delivered
"city": "Rome", // name of the city
"region": "Italy", 
"location": { // coordinates
    "lat": 41.895466,
    "lng": 12.482324
},
"details": // this is a list of pictures and google maps links, we won't need these

The structure is pretty simple, the only problem I saw is that those timestamps are stuck in 2019, so we will need to account for this in the script.

Since it would be really wasteful to make a request to that URL every time we are going to run our script, we can download the JSON file and put it in the same directory as the script.

Let’s now start writing the program, first of all we will need a couple of imports from the Python standard library:

import json
import time
from datetime import datetime, timezone

We need the json library to decode the JSON file and the time and datetime libraries to get the current time and do some “space-time bending” stuff to account for the mismatching date in the JSON.

To make the code a bit more readable we can define two constants:

JSON_TIMESTAMP_YEAR = 2019
FILENAME = "route.json"

JSON_TIMESTAMP_YEAR actually matches the year of the timestamps inside the JSON, and the FILENAME constant is just the path for the JSON file.

Now we are going to write a function that will change the current UNIX epoch timestamp, so that the time (hours, minutes and seconds) is the same but the date will be Christmas eve of 2019:

def get_christmas_eve_timestamp(year: int) -> int:
    now_utc = datetime.now(timezone.utc)
    target_date = now_utc.replace(year=year, month=12, day=24)
    return int(target_date.timestamp()) * 1000

You may notice in the last line that we multiply the timestamp times 1000, that’s because the .timestamp() method returns a float with the integer part equals to the number of seconds since January 1st 1970 and the real part is the amount of milliseconds that passed since that date, and the JSON file actually includes the milliseconds in the integer part of the timestamp.

Note: from now on I will refer to the current timestamp as the return value from this function.

Of course we need to parse the JSON file, so here is another helper function to do just that:

def read_route_json() -> list:
    with open(FILENAME, "r") as file:
        dict = json.load(file)
        return dict["destinations"]

We parse the document with json.load() and then we return the only entry in the file, the destinations list.

With these two helper functions out of the way we can focus on the logic of this application, during the lifetime of the application we can have 4 cases:

  • Santa has not started his trip yet, so the current timestamp is lower than the first arrival timestamp
  • Santa is delivering gifts to a certain city, so the current timestamp is in range of one of the JSON object’s arrival and departure timestamp
  • Santa is moving between cities, the current timestamp is in range of a object’s departure timestamp and the the next object’s arrival timestamp
  • Santa has finished his trip, the current timestamp is greater than the last object’s departure timestamp

So with that in mind we can write a function that will return a string with Santa’s status given the route and the current timestamp:

def get_santa_status(current_time, route) -> str:
    # Case 1: Santa hasn't started
    if current_time < route[0]["arrival"]:
        return "Santa is sleeping at the North Pole."

    for i in range(len(route)):
        stop = route[i]

        # Case 2: Santa is delivering present at a city
        if stop["arrival"] <= current_time <= stop["departure"]:
            return f"🎁 DELIVERING PRESENTS! Location: {stop['city']} ({stop['region']}) ({stop['location']['lat']}, {stop['location']['lng']})"

        # if statement needed to avoid overflow
        if i < len(route) - 1:
            next_stop = route[i + 1]
            # Case 3: Santa is travelling
            if stop["departure"] < current_time < next_stop["arrival"]:
                return f"🛷 FLYING! From {stop['city']} ({stop['region']}) -> to {next_stop['city']} ({next_stop['region']})\n"

    # Case 4: Santa has finished
    return "🎅 Santa has finished his run! Back at North Pole."

Finally we need to put everything together, we will have a loop that checks for updates every 30 seconds, since from what I saw the minimum distance between timestamps is 1 minute:

def main():
    route_data = read_route_json()

    try:
        # clear screen with ANSI codes:
        print("\033[H\033[J", end="")
        while True:
            current_time = get_christmas_eve_timestamp(JSON_TIMESTAMP_YEAR)

            status = get_santa_status(current_time, route_data)
            print(status)

            time.sleep(30)

            # clear screen with ANSI codes:
            print("\033[H\033[J", end="")

    except KeyboardInterrupt:
        print("Tracking stopped.")

if __name__ == "__main__":
    main()

To make the UI a bit cleaner I added those two print statement with seemingly garbage strings, those are ANSI codes and are interpreted by the terminal as escape sequences to clear the screen, and the try-except statement so that when the user stops the execution with Ctrl+C we don’t show an error but just an exit message.

If you want to just see how it works without copying the code from here, this is the project repository, you just need to clone it and run the script with python main.py.
