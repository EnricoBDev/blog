Conditional logic in MIPS assembly⌗

To build traditional conditional logic the language offers us some control flow instructions and some conditional data transfer instructions, let’s see them

Mnemonic Sintax Meaning
beq (branch equal) beq $t0 $t1 Label Jumps to “Label” if $t0 and $t1 are equal
bne (branch not equal)* bne $t0 $t1 Label Jumps to “Label” if $t0 and $t1 are different
slt (set on less than)* slt $s0 $t0 $t1 Sets register $s0 to 1 if $t0 < $t1
j (jump) jump Label Jumps to Label

* there is also another variant of this instruction that compares the content of a register and an inmmediate value (slti)

Now we will see every condition present in any modern programming language, in each example we will translate the same if statement, but we will change the condition, here is the C code we will translate:

if (f condition g){
    f +=1;
} else {
    f -= 1;
}

We will also suppose that the variable f is the $t0 register and g is $t1

!= and == condition⌗

To create these conditions we just need to use the provided bne, beq instructions, let’s see an example with the != condition:

beq $t0, $t1, Else
addi $t0, $t0, 1
j Exit

Else:
    subi $t0, $t0, 1

Exit:

We can see that in this example we actually use the opposite instruction (we are using beq for a != condition) but it still works, since we jump to the else label if the two variables are equal, otherwise we continue with the block inside the if statement.

>= condition⌗

To make this condition we are going to use the slt instruction and we are going to use the same trick as before, where we negate the condition and jump to Else if the negated condition is true, let’s see:

slt $s0, $t0, $t1
bne $s0, $zero, Else
addi $t0, $t0, 1
j Exit

Else:
    subi $t0, $t0, 1

Exit:

slt $s0, $t0, $t1 is the most important line, it exploits the fact that the negated condition of < is >=, so if the condition we want to achieve is true slt sets $s0 = 0, then after that we use the trick i just told you in the previous paragraph where we jump to Else if the negated condition is true.

< condition⌗

To obtain this condition we just need to change bne to beq in the previous snippet since now if the condition is true the output from slt will be 1, after all slt checks the < condition

slt $s0, $t0, $t1
beq $s0, $zero, Else
addi $t0, $t0, 1
j Exit

Else:
    subi $t0, $t0, 1

Exit:

> condition⌗

To translate this condition we use the fact that by inverting the operands of a < condition we get the > condition, let’s see the implementation:

slt $s0, $t1, $t0
beq $s0, $zero, Else
addi $t0, $t0, 1
j Exit

Else:
    subi $t0, $t0, 1

Exit:

<= condition⌗

The last condition is <=, to translate it we exploit the fact that <= is the negation of >:

slt $s0, $t1, $t0
bne $s0, $zero, Else
addi $t0, $t0, 1
j Exit

Else:
    subi $t0, $t0, 1

Exit:

We use the same check slt $s0, $t1, $t0 that we used for > but the difference is that if the condition we want to achieve is true, then slt outputs a 0 (as we said <= is the negation of >, and we just switched the operands when using slt)

And there you have it, these are all the traditional logic conditions you are familiar with, now a couple of thoughts:

  • pseudoinstructions that do these conditions (for example bgt for >) exists, these are instructions that are translated to other more basic instructions by the assembler (that’s what the $at register is used for)
  • these translations can be quite challenging to remember (and i really hope that my explanation was not too short or confusing), but recreating them could be a good exercise to learn or practice another assembly flavour (even though the ISA may give you different instructions)
